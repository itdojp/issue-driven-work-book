# 第6章：進捗報告の定型（現状/次/ブロッカー/期限）

## この章で学ぶこと

- 進捗は“短文・高頻度・ブロッカー提示”で報告する
- ブロッカーは早期に表面化させる
- 期限と次アクションを明確化する

## 成果物（書けるようになるもの）

- 進捗報告テンプレを使える（付録: [進捗報告](../../appendices/templates/status-report/)）
- 支援依頼を具体的に書ける

## 本文

進捗は“頑張っている”ではなく、“いま何が終わっていて次に何をするか”を伝える。

### 進捗報告の最小

- 現状: 完了/未完了
- 次: 次にやること
- ブロッカー: 何が詰まっているか
- 期限: いつまでに
- 支援依頼: 何が欲しいか

## 具体例（悪い例→良い例）

### 悪い例

```md
順調です。頑張っています。
ブロッカーや期限がない
```

### 良い例

```md
現状: バリデーション実装完了、文言は未着手
次: 文言案を作りレビュー依頼
ブロッカー: 文言ポリシー未確定
期限: 2026-02-28
支援: 文言ポリシーの決定（レビュー30分）
```

## チェックリスト

- [ ] 現状/次/ブロッカー/期限が揃っている
- [ ] 支援依頼が具体的
- [ ] 短文で状況が伝わる

## まとめ

- テンプレと受け入れ条件で“期待する結果”を固定し、手戻りを減らす
- 記録（ログ/証跡）を残し、レビュー可能な状態にする
