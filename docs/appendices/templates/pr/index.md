# PRテンプレ

## 何を

- 変更点

## なぜ

- 背景/目的

## 影響

- 影響範囲
- リスク

## テスト

- 実施内容（未実施の場合は理由と代替の確認方法を記載）

## レビュー観点（任意）

- 見てほしい順序/注目点

## ロールバック

- 戻し方

## 関連Issue

- （記入）

## 記入例（抜粋）

```md
## 何を

- 入力フォームの必須項目チェックを追加し、エラー文言を具体化

## なぜ

- 直近2週間で「未入力のまま保存」起因の問い合わせが増加（誤操作率 5%→1% 以下へ低減したい）

## 影響

- 影響範囲: 入力フォームの保存処理（API/DBスキーマ変更なし）
- リスク: 既存の自動入力が弾かれる可能性（境界条件をテストで確認）

## テスト

- 単体: 必須項目/境界値のバリデーション
- E2E: 主要導線（作成→保存→エラー表示→再保存）

## レビュー観点

- 破壊的変更が無いこと（API/DB）
- エラー表示の文言と運用影響

## ロールバック

- Feature flag をOFFにする（設定手順をRunbookに記載）

## 関連Issue

- #123
```
