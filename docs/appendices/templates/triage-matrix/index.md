# トリアージ判断表（影響度×緊急度）

| 影響度 \ 緊急度 | 高 | 中 | 低 |
|---|---|---|---|
| 高 | 即時対応（P0/P1） | 優先対応（P1） | 予定化（P2） |
| 中 | 優先対応（P1） | 予定化（P2） | 低優先（P3） |
| 低 | 予定化（P2） | 低優先（P3） | 対応しない選択も可 |

- 影響度: 利用者/売上/セキュリティ/法令の観点で評価
- 緊急度: 期限/依存/時間経過で悪化するかで評価

## P0〜P3 運用補足（目安）

- P0: 全面停止/重大事故の疑い。直ちに着手し、短い間隔で状況更新する
- P1: 主要影響。優先対応としてタスクを割り当て、復旧方針と期限を明確にする
- P2: 予定化して計画的に対応する（スプリント/期日等に組み込む）
- P3: 低優先としてバックログ化し、定期的に見直す（放置ではなく“管理”する）

## 「対応しない」判断の記録（最小）

- 理由（なぜやらないか）
- 代替案（回避策/運用で吸収する等）
- 再オープン条件（影響拡大、頻度増加、依存の発生等）
