# チェックリスト集

## 起票（Issue作成）

- [ ] 背景（現状/問題）を 1〜3 行で書いた
- [ ] 目的（何を変えるか）を明記した
- [ ] 受け入れ条件（Acceptance Criteria）をチェックリストで書いた
- [ ] スコープ/非スコープを分けた
- [ ] 影響範囲（ユーザー/システム/運用）を整理した
- [ ] 依存（他 Issue/外部要因）とリスクを記載した
- [ ] 関係者（Owner/Reviewer/窓口）を明確にした

## トリアージ（優先度決定）

- [ ] 影響度（顧客/売上/セキュリティ/法令）を評価した
- [ ] 緊急度（期限/依存/時間経過で悪化）を評価した
- [ ] 次アクション（いま対応/予定化/やらない）を合意した

## 調査（調査ログ）

- [ ] 事象の再現条件（環境/手順/入力）を固定した
- [ ] 期待結果と実際結果を並べた
- [ ] 証跡（ログ/スクリーンショット/設定差分）を残した
- [ ] 試行と結果（何をやってどうなったか）を時系列で残した
- [ ] 一次情報（公式 docs/仕様/実装）に到達した
- [ ] 代替案/リスク（採用しない案も含む）を記録した

## 分割（縦切り）

- [ ] 価値が出る単位で縦に分割した
- [ ] 依存関係が少ない順に並べ替えた
- [ ] MVP（最小リリース）を決めた
- [ ] DoR/DoD を明文化した（付録テンプレ参照）
- [ ] 見積もり/期日が必要なら根拠と前提を残した

## PR（レビュー可能性）

- [ ] 変更点を 1 段落で要約した
- [ ] 影響範囲（機能/データ/互換性）を明記した
- [ ] テスト観点/実施結果を添付した
- [ ] ロールバック手順/切り戻し条件がある
- [ ] 運用反映（Runbook/FAQ/ADR/手順書）への影響を記載した
- [ ] レビュー手順（見てほしい順序/観点）を提示した

## 完了（クローズ）

- [ ] 受け入れ条件を満たした
- [ ] 関連 Issue/PR/リリースを相互リンクした
- [ ] ナレッジ化（Runbook/FAQ/ADR/手順書）に転記した
- [ ] 監視/メトリクス/アラートの変更があれば反映した
- [ ] クローズ理由が説明できる（やらない選択を含む）
