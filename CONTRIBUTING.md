# コントリビューションガイド

## 方針

- 誤字脱字・表現の改善
- 事実関係/技術的誤りの指摘
- 参照情報の追加（一次情報優先）
- 読者の前提に対する補足（初学者向けの説明）

## 手順（最小）

1. Issue を作成し、背景/目的/期待する結果を記載する
2. Fork してブランチを作成する
3. 変更を加え、`npm test`（lint/link）を通す
4. Pull Request を作成し、変更点/影響範囲/確認方法を記載する

## ライセンス

コントリビューションを行うことで、当該変更が CC BY-NC-SA 4.0 の条件で提供されることに同意したものとみなします。
